# /workflows-compound

记录最近解决的问题，积累团队知识。

## 目的

在上下文还清晰时捕获问题解决方案，创建结构化文档供未来参考。

**为什么叫"compound"（复利）？** 每个记录的解决方案都会让团队知识产生复利效应。第一次解决问题需要研究。记录下来后，下次遇到同样问题只需几分钟。

## 捕获内容

- **问题症状**：具体错误信息、可观察的行为
- **尝试过的排查步骤**：什么方法没用以及原因
- **根因分析**：技术层面的解释
- **有效解决方案**：带代码示例的分步修复
- **预防策略**：如何避免未来再次发生
- **交叉引用**：相关问题和文档的链接

## 执行规则（必须遵守）

执行此命令时，**必须**使用 Task 工具启动子代理来完成文档：

### 步骤 1：启动子代理（并行）

使用 Task 工具启动 `technical-writer` 子代理，提供以下信息：

```
在 docs/compound/{category}/ 目录下创建问题文档。

问题描述: {从对话历史提取}

需要包含：
1. 问题症状 - 具体错误信息或行为描述
2. 根因分析 - 技术层面的原因解释  
3. 解决方案 - 代码示例
4. 涉及文件 - 文件列表
5. 预防策略 - 如何避免类似问题
6. 标签 - 相关标签

分类选择: {根据问题类型选择 Output Categories}
```

### 步骤 2：确认完成

子代理完成后，更新 TODO 列表标记文档已创建。

## 复利哲学

```
Plan -> Work -> Review -> Compound -> Repeat
```

每个循环都在积累：计划为未来的计划提供参考，审查发现更多问题，模式被记录下来。

**每一单位的工程工作都应该让后续工作变得更容易——而不是更难。**
